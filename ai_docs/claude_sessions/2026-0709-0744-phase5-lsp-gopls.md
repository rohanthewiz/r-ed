## `internal/lsp` — the client (new package)

### types.go
- Wire structs for the protocol subset only: Position/Range/Location,
  Diagnostic (+severity consts), didOpen/didChange/didSave/didClose
  params, TextDocumentPositionParams, Hover.
- `Hover.HoverText()` flattens the three shapes servers actually send
  (MarkupContent, bare MarkedString, arrays of either).
- **URI conversion** via net/url (`PathToURI`/`URIToPath`) — spaces
  escape correctly; non-file schemes (zipfile: etc.) return "" and
  degrade to a no-op.
- **UTF-16 ↔ rune columns** (`UTF16Col`/`RuneCol`) — LSP counts UTF-16
  code units, the Buffer counts runes. Real bug caught by tests: a
  column landing on the *second* unit of a surrogate pair must resolve
  to that rune's index (`utf16Col < col+RuneLen(r)`, not `col >=`).

### client.go
- Content-Length framing over stdio; `Call` (id-correlated, 5s
  timeout), `Notify`, both goroutine-safe (write mutex).
- Read loop dispatches: responses → pending channel; notifications →
  `onNotify` callback (runs ON the read-loop goroutine — callers must
  post events, never touch state); server→client **requests get
  auto-replies** — `workspace/configuration` echoes one empty object
  per item (gopls blocks type-checking on this!), everything else
  gets null.
- Pipe close fails all pending Calls promptly + fires `onExit` once.
- `Close()`: fire-and-forget `exit` notification, close stdin, reap on
  a goroutine with a 2s kill backstop — never blocks the caller.
- `Initialize()` advertises minimal capabilities (didSave sync,
  plaintext-preferred hover). didChange always sends ONE rangeless
  change event = full-document replace — no incremental bookkeeping.
- `Definition()` normalises Location | []Location | null → slice.

## `internal/app/lsp.go` — the manager

- `App.lsp lspState` — client (behind an `lspConn` interface for
  fakes), starting/dead flags, per-path version + syncedRev maps,
  debounce timers, diags map, navStack. **Main-loop-only mutation**;
  six new custom events do the goroutine bridging: lspReadyEvent,
  lspExitEvent, lspDiagsEvent, lspSyncEvent, lspDefinitionEvent,
  lspHoverEvent.
- **Async start**: first opened `.go` file → LookPath("gopls") (miss =
  dead flag, silent, same contract as formatters) → spawn + initialize
  on a goroutine → lspReadyEvent → `handleLSPReady` didOpens every
  already-open Go tab (docs opened mid-handshake are never lost).
  `handleLSPReady` also guards the dead-after-post race (closes the
  late client instead of resurrecting).
- **Sync via `Tab.EditRev`** (new field): every content mutation bumps
  it — the 5 tab.go mutators, undo's applySnapshot, ToggleLineComment,
  Reload. Cursor motion deliberately does NOT. `lspAfterEvent()` runs
  after every handleEvent dispatch, compares EditRev vs syncedRev,
  (re)arms a per-path 300ms `time.AfterFunc` → lspSyncEvent → one
  full-text didChange with version++. **Saves flush pending changes
  before didSave** so the server never diagnoses stale text.
- Server death: mark dead, clear ALL diags (nothing left to retract
  stale squiggles), no auto-restart (would flap).
- **Diagnostics**: stored per absolute path (gopls publishes for
  non-open files too — kept, cheap). `lspDiagSource` (registered after
  gitDiffSource in openFile so the diag dot outranks the git mark)
  emits underline+FG spans per diagnostic and one '●' gutter mark per
  line with worst severity winning. Stale ranges past EOF cull. Zero-
  width ranges stretch one rune right. Status bar: " · ✗ 2 ⚠ 1 ℹ 1"
  suffix on the left segment, empty when clean.
- **Definition**: request on a goroutine (after flushing changes);
  handler opens the target, pushes origin on a 50-deep navStack,
  moves cursor via UTF-16→rune conversion. Esc-o / "Jump back" pops.
- **Hover**: flattened via `hoverLines` (strips ``` fences, trims
  blank edges, caps at 12 lines + "…"); response for a tab the user
  already left is dropped.
- Menu: new **Code group** after Git — Go to definition / Hover info /
  Jump back (predicates hasLSPActions / hasNavBack). Leaders:
  **Esc-d / Esc-i / Esc-o**.
- Theme: new DiagError / DiagWarning / DiagInfo (Tokyo Night
  red/amber/blue), added to the tripwire test.

### hovermodal.go
- `hoverModal` implements the standard modal interface but anchors to
  the caret via new `Tab.CursorScreenCell(w,h)` (exported; same math
  as Render's hardware cursor — clamps like Render on short lines).
  Below the caret by default, flips above near the bottom, centered
  fallback when the caret is scrolled away. ANY key dismisses; clicks
  dismiss; wheel/motion do NOT (user may be reading). Long lines
  truncate with "…", max width 66.
